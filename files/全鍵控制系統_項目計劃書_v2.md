# 全鍵控制系統 - 項目計劃書 (v2.0 可擴展版)

**項目名稱：** KeyMaster - 全鍵控制應用啟動器  
**版本：** v1.0 (MVP → v2.0+)  
**目標平台：** Windows 10+ / Linux (GTK/Wayland) / macOS  
**預計完成時間：** 6-8 個月  
**開發團隊規模：** 1-2 人 (全職) 或 1 人 (兼職 6 個月)

---

## 📋 項目概述

### 願景
創建一個「開發者友好的、全鍵盤控制的應用啟動器和系統控制中樞」，讓用戶在 95% 的開發工作中無需使用鼠標或觸控板，提高工作效率 2-3 倍。**架構應支持長期擴展，允許社群通過插件、主題、快捷鍵宏等定制系統。**

### 核心特性（MVP）
1. **快速應用啟動** - Win+K 打開，模糊搜索應用，支持別名
2. **終端浮窗** - Ctrl+Alt+T 打開獨立終端，不影響後台應用
3. **鍵盤鼠標控制** - 用方向鍵/WASD 控制光標，精準點擊
4. **直接命令執行** - Win+Shift+P 執行系統命令，無需打開終端
5. **熱鍵配置面板** - 內置快捷鍵管理，自動衝突檢測

### 成功指標
- MVP 發佈後，開發人員日均使用 6+ 小時
- 鍵盤操作佔 90%+ (相比原來的 40%)
- 應用啟動平均延遲 <100ms
- 用戶保留率 >80% (前 3 個月)
- **v1.0+ 支持自訂主題、快捷鍵宏、社群插件**

## 🎯 可擴展性設計原則

本項目採用「**4層可擴展架構**」，確保每個階段的功能都能輕鬆擴展，無需返工：

| 層級 | 名稱 | 用途 | v1.0 狀態 | 擴展方向 |
|-----|------|------|---------|---------|
| **1** | **命令路由層** | 統一的命令分發 | CommandRouter | 支持官方+社群命令 |
| **2** | **事件發射層** | 實時雙向通訊 | 內置事件總線 | 插件事件廣播 |
| **3** | **分層配置系統** | 用戶定制點 | 結構化 TOML | 主題、宏、插件配置 |
| **4** | **統一索引層** | 通用數據搜索 | 文件+應用索引 | 歷史、剪貼簿、插件數據 |

### 為什麼這樣設計？
- **CommandRouter** → v3.0 插件系統無需改核心代碼
- **事件層** → 終端、搜索進度、通知都能實時推送
- **配置系統** → 用戶主題、快捷鍵宏、插件設置無需硬改
- **統一索引** → 新增搜索源（剪貼簿、歷史）時只需添加 indexer，無需重構

```
前端：
  - Prettier 自動格式化
  - ESLint 靜態檢查
  - 命名：camelCase
  - 組件結構：Presentational + Container

後端：
  - Rust fmt (cargo fmt)
  - Clippy 檢查 (cargo clippy)
  - 命名：snake_case
  - 函數長度：<100 行
  - 新功能必須實現 Command trait（如適用）
```

### Git 工作流
```
分支模型：
  main: 穩定版本，打 tag 發佈
  develop: 開發主分支
  feature/*: 功能分支
  bugfix/*: 修復分支
  arch/*: 架構改進分支

提交信息格式：
  [type]: [description]
  
  type: feat, fix, docs, style, refactor, test, chore, arch
  示例：
    feat: add clipboard history with image support
    arch: implement CommandRouter for plugin extensibility
    feat: add EventBus for real-time updates
```
