## 🏗️ 分層架構設計

### 1️⃣ **演示層 (Presentation Layer)**

```
React 前端 (src/)
├── UI 組件層
│   ├── CommandPalette (主交互界面)
│   ├── Terminal (終端視圖)
│   ├── FloatingWindow (浮窗容器)
│   └── 各功能子組件
├── 狀態管理層 (Zustand)
│   ├── appStore (應用全局狀態)
│   ├── hotkeyStore (熱鍵配置)
│   └── uiStore (UI 狀態)
├── Hook 層 (業務邏輯)
│   ├── useIPC (RPC 通訊)
│   ├── useSearch (搜索邏輯)
│   └── useTerminal (終端管理)
└── 通訊層 (services/)
    └── invoke() 包裝層 → 後端 RPC 命令
```

### 2️⃣ **IPC 通訊層 (Bridge)**

```
Tauri IPC 橋接
├── 前端: @tauri-apps/api invoke()
└── 後端: #[tauri::command] 宏

命令模式：
  前端調用: invoke('command_name', { params })
  後端響應: async fn command_name() -> Result<Response>
  錯誤處理: 統一的 Rust Result + TypeScript Error
```

### 3️⃣ **業務邏輯層 (Service Layer)**

```
Rust Managers (src-tauri/src/managers/)
├── HotkeyManager
│   ├── 全局熱鍵監聽 (rdev)
│   ├── 熱鍵路由和分派
│   ├── 衝突檢測
│   └── 配置持久化
├── AppManager
│   ├── 應用列表掃描 (Registry / .desktop)
│   ├── 應用搜索和別名匹配
│   ├── 應用啟動 (CreateProcess / spawn)
│   └── 最近應用記錄
├── TerminalManager
│   ├── PTY 進程創建和管理
│   ├── 輸入/輸出處理
│   ├── 進程生命週期監控
│   └── 終端尺寸同步
├── MouseManager
│   ├── 光標位置查詢
│   ├── 光標移動邏輯 (加速度曲線)
│   ├── 點擊模擬
│   └── 精度配置
├── SearchManager
│   ├── 文件索引 (tantivy)
│   ├── 模糊搜索
│   ├── 結果排序和緩存
│   └── 索引增量更新
└── ConfigManager
    ├── 配置文件 I/O (TOML)
    ├── 默認配置和迁移
    └── 實時配置熱加載
```

Rust 數據結構 (models/)

## 🔄 數據流和交互流程

### 流程 1: 應用啟動

```
1. 用戶按 Win+K
   ↓
2. HotkeyManager 監聽到事件
   ↓
3. 路由到 "launcher" 命令
   ↓
4. Tauri IPC 通知前端
   ↓
5. React 顯示 CommandPalette
   ↓
6. AppManager 在後台掃描應用列表
   ↓
7. 應用列表返回前端
   ↓
8. 用戶搜索應用名稱
   ↓
9. 前端實時模糊搜索 (fuse.js)
   ↓
10. 用戶按 Enter
   ↓
11. invoke('launch_app', { app_path })
   ↓
12. AppManager 執行 CreateProcess / spawn
   ↓
13. 應用啟動，窗口獲焦
```

### 流程 2: 終端浮窗

```
1. 用戶按 Ctrl+Alt+T
   ↓
2. HotkeyManager 路由到 "terminal" 命令
   ↓
3. Tauri 通知前端顯示終端浮窗
   ↓
4. 前端: <Terminal /> 組件掛載
   ↓
5. 前端: invoke('spawn_terminal')
   ↓
6. TerminalManager 創建 PTY (openpty)
   ↓
7. 啟動 shell 進程 (bash / zsh / cmd)
   ↓
8. 返回 PTY 文件描述符給前端
   ↓
9. 前端: xterm.js 連接到 PTY
   ↓
10. 用戶輸入 → PTY 寫入 → shell 執行 → 輸出讀取 → xterm 顯示
```

### 流程 3: 鍵盤鼠標控制

```
1. 用戶按方向鍵 (↑)
   ↓
2. HotkeyManager 監聽
   ↓
3. invoke('move_mouse', { dx: 0, dy: -10 })
   ↓
4. MouseManager 計算新位置 (考慮加速度)
   ↓
5. 調用平台 API 移動光標
   └─ Windows: SetCursorPos()
   └─ Linux: xdotool mousemove
   └─ macOS: CGWarpMouseCursorPosition()
   ↓
6. 返回新光標位置給前端 (用於 UI 顯示)
   ↓
7. 前端更新光標視覺反饋
```

### 流程 4: 熱鍵配置更新

```
1. 用戶在 HotkeyConfig UI 修改快捷鍵
   ↓
2. 前端: invoke('update_hotkey', { oldKey, newKey, command })
   ↓
3. HotkeyManager 檢查衝突
   ↓
4. 如有衝突，返回錯誤 + 建議
   ↓
5. 如無衝突，更新內存配置
   ↓
6. ConfigManager 寫入 hotkey.toml
   ↓
7. HotkeyManager 重新註冊系統熱鍵
   ↓
8. 返回成功給前端，顯示確認消息
```

## 🔐 全局應用狀態

```rust
// src-tauri/src/state.rs

pub struct AppState {
    pub hotkey_manager: Arc<Mutex<HotkeyManager>>,
    pub app_manager: Arc<Mutex<AppManager>>,
    pub terminal_manager: Arc<Mutex<TerminalManager>>,
    pub mouse_manager: Arc<Mutex<MouseManager>>,
    pub search_manager: Arc<Mutex<SearchManager>>,
    pub config_manager: Arc<Mutex<ConfigManager>>,
    pub logger: Arc<Logger>,
}

// 在 main.rs 中初始化：
pub fn create_app_state() -> AppState {
    AppState {
        hotkey_manager: Arc::new(Mutex::new(HotkeyManager::new())),
        app_manager: Arc::new(Mutex::new(AppManager::new())),
        terminal_manager: Arc::new(Mutex::new(TerminalManager::new())),
        mouse_manager: Arc::new(Mutex::new(MouseManager::new())),
        search_manager: Arc::new(Mutex::new(SearchManager::new())),
        config_manager: Arc::new(Mutex::new(ConfigManager::new())),
        logger: Arc::new(Logger::new()),
    }
}

// 在 Tauri 中注冊：
.manage(create_app_state())
```

## 🔌 插件系統架構 (v3.0+)

```rust
pub trait Plugin: Send + Sync {
    fn name(&self) -> String;
    fn version(&self) -> String;
    fn on_command(&self, cmd: &str, args: &[String]) -> PluginResult;
    fn on_hotkey(&self, key: &str) -> bool;
    fn get_commands(&self) -> Vec<String>;
}
```

### 插件加載機制

```
plugins/ 目錄結構
├── official/
│   ├── calculator/
│   │   ├── plugin.toml
│   │   └── lib.rs
│   └── clipboard/
└── community/
    └── (用戶提交的插件)

插件加載流程：
1. 掃描 plugins/ 目錄
2. 讀取 plugin.toml 元數據
3. 動態加載 .so / .dll / .dylib
4. 調用 on_load() 初始化
5. 註冊命令到命令路由器
```
